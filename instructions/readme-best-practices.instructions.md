## README Structure

Every project README should have:

1. **Title + one-line description** — what this project does.
2. **Badges** (optional) — build status, coverage, version.
3. **Quick start** — how to build/install/run in ≤5 commands.
4. **Prerequisites** — compiler version, dependencies, OS.
5. **Usage** — basic examples with actual commands/code.
6. **Project structure** (if non-trivial) — brief tree of important dirs.
7. **Contributing** — how to contribute, code style, PR process.
8. **License** — always specify.

## Guidelines

- Keep it concise. Link to `docs/` for deep dives. No walls of text.
- Use fenced code blocks with language tags for all commands and code.
- Show actual output where it helps comprehension.
- Keep prerequisites section honest — list exact versions tested.
- Update README when adding/removing features or changing build steps.
